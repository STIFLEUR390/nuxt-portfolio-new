## Anti-Patterns Verdict

**CAUTIOUSLY NOT AI-GENERATED — BUT RISKS SLIDING THERE**

**LLM assessment**: The architecture and many details feel human and deliberate (View Transition API, FAQ tabs, data-driven structure, PolaroidItem). But the portfolio suffers from content-level tells that collectively undermine trust.

**Specific tells**:
- Unsplash headshot as profile photo — stock face on a freelance portfolio
- Zero actual project screenshots — gradient placeholders instead of real work
- Blog posts with all Unsplash images, no original diagrams or screenshots
- Services with overlapping content (APIs vs WhatsApp both mention Stripe/Flutterwave)
- Testimonials use anonymous "Client X" identifiers

## Overall Impression

Solid technical foundation undermined by content that doesn't match the craft. The code is cleaner than most portfolios I review — typed data files, proper tokens, good a11y baseline. But the portfolio fails at its primary job: proving the developer builds real products. Gradient placeholders, stock photos, and overlapping service descriptions tell a recruiters' Spidey-sense story that the code can't fix alone.

**Single biggest opportunity**: Replace stock/gradient images with real project screenshots. This alone would move the score 5+ points.

## What's Working

**S1: View Transition API theme switch** — `ColorModeButton.vue` clip-path reveal is a genuine custom touch that demonstrates real frontend depth. Most portfolios don't have this.

**S2: Clean modular architecture** — All content in typed data files, pages as thin templates, single-responsibility components. Professional-grade structure that would scale to a larger project.

**S3: FAQ accordion + tabs** — Best interactive component on the site. Tab-categorized, markdown-rendered, keyboard-navigable, plus-to-cross icon rotation. Nuxt UI composition done right.

**S4: Accessibility baseline** — Skip link, safe-area support, custom selection colors, carousel ARIA labels, `MotionConfig :reduced-motion="'user'"`. Someone actively cared about this.

## Priority Issues

**P0 — Hero display font (Instrument Serif) not applied**
- **Where**: `landing/Hero.vue` title renders without `font-serif` class
- **Why**: DESIGN.md §3 specifies Instrument Serif for hero display only. Without it, the typographic identity is invisible — Public Sans everywhere reads as generic.
- **Fix**: Add `class="font-serif"` to Hero title slot wrapper
- **Suggested command**: `$impeccable typeset`

**P0 — Generic Unsplash photo as profile avatar**
- **Where**: `app.config.ts:4-5`
- **Why**: Stock face on a freelance portfolio is the #1 trust killer. Founders and recruiters immediately question "is this real?"
- **Fix**: Replace with actual photo, or use monogram avatar as honest fallback
- **Suggested command**: `$impeccable delight`

**P0 — Project cards show gradient placeholders, not real screenshots**
- **Where**: `ProjectImage.vue:10-25`
- **Why**: A product portfolio with zero product images. The design IS the product; the work is invisible.
- **Fix**: Render `project.image` as `<img>` with gradient fallback
- **Suggested command**: `$impeccable delight`

**P1 — Duplicate `titleTemplate` + name mismatch**
- **Where**: `app.vue:29` and `default.vue:19` both set `titleTemplate`. "Hérold H." vs "Franck Hérold TAMTO TAMKO".
- **Why**: Confusing for SEO and user trust. Which name is correct?
- **Fix**: Remove duplicate, standardize on full name everywhere
- **Suggested command**: `$impeccable harden`

**P1 — Blog detail page bypasses layout**
- **Where**: `blog/[...slug].vue:58` uses `<UMain class="mt-20 px-2">` with hardcoded margins
- **Why**: Inconsistent with all other pages. Maintenance burden.
- **Fix**: Use standard `UPage` + `UPageHero`/`UPageSection` pattern
- **Suggested command**: `$impeccable layout`

**P2 — No search despite MeiliSearch blog post**
- **Where**: Blog post mentions search integration, no search UI exists
- **Why**: Technical evaluators will notice the gap between claims and delivery
- **Fix**: Implement search or defer the blog post

**P2 — 15% blue coverage rule potentially violated**
- **Where**: Cumulative `bg-primary/5` + `border-primary/10` surfaces across sections
- **Why**: DESIGN.md states strict 15% rule for blue. Tinted surfaces add up.
- **Fix**: Audit total blue surface area; use `surface-subtle` for secondary cards

## Minor Observations

1. `-mx-4` negative margins in Testimonials carousel — works but fragile. Consider `overflow-visible`.
2. `!important` in `pt-0!` across 8+ components — maintenance smell. Better: zero default in app.config.
3. `#474747` dark mode selection color undocumented — add to DESIGN.md or align with palette.
4. `10px` font-size in ProjectImage tags — off the type ramp. Use `text-xs` (12px) instead.
5. `0.875rem` in skip-link font-size — off the type ramp. Use 1rem (body) or 0.8125rem (label).
6. Error page has standalone `UToaster` — `UApp` already provides toast. Double-toast risk.
7. Both PolaroidItem images on about page get `-rotate-3` (index 0 and 1, even/odd = 0=even, 1=odd → 0: -rotate-3, 1: rotate-3). Works as intended.
8. `htmlAttrs: { lang: 'fr' }` hardcoded — i18n supports EN, should be dynamic.
